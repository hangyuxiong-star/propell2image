# propell2image 项目文档

## 项目简介

propell2image 是一个对话式多模态 AI 内容生成系统，将 LangGraph ReAct Agent 与专业生成工具链深度整合——图片生成（火山引擎 Seedream 5.0 / Qwen-Image）、视频生成（火山引擎 Seedance 1.5 Pro）、3D 模型生成（腾讯混元 AI3D）、语音合成（阿里 Qwen-TTS）、多模态理解（Qwen3-Omni-Flash）。用户在聊天界面通过自然语言描述创意需求，Agent 自动拆解任务、串联 20 个专用工具、交付端到端成品并自动插入无限画布。系统采用 FastAPI + React + TypeScript + Excalidraw 架构，支持 SSE 流式对话、WebSocket 画布实时同步、Skill 渐进式扩展和跨对话工作区记忆。

| 核心能力 | 说明 |
|---|---|
| Agent 编排引擎 | LangGraph ReAct 架构，20 个工具注册，最大递归 200 步，双 LLM 提供商热切换 |
| 流式对话协议 | SSE 实时推送 delta / tool_call / tool_result 事件，JSON 增量参数解析 |
| 多模态内容生成 | 图片 (8 种宽高比)、视频 (3 种模式)、3D 模型 (OBJ/GLB)、TTS (10 种语言)、BGM 混音 |
| Skill 扩展系统 | Progressive Loading 按需加载，6 个内置 Skill，custom/ 热加载自定义 |
| 工作区记忆 | 6 个身份文件注入 System Prompt，Agent 自主维护跨对话记忆 |
| 无限画布 | Excalidraw 矢量画布，自动布局、3D 预览、多客户端 WebSocket 同步 |

### Agent 核心循环流程

```
用户输入 (Natural Language)
    │
    ▼
System Prompt 组装 (4 部分动态拼装)
    ├── 基础角色定义 (SYSTEM_PROMPT)
    ├── 动态工具列表 (tools_list_text)
    ├── Skills 上下文块 (Progressive Loading 元数据)
    └── Workspace 身份记忆 (有实质内容的文件)
    │
    ▼
create_react_agent(model, tools, prompt)
    │
    ▼
agent.astream({"messages": [...]}, stream_mode=["messages"])
    │
    ├── AI 思考: AIMessageChunk → delta 事件 (文本增量)
    │
    ├── 工具调用: tool_call_chunks → 参数累加 → JSON 解析
    │       │
    │       ├── generate_volcano_image(文本, 尺寸)      → 图片 URL
    │       ├── edit_volcano_image(图片, 文本)          → 编辑后图片
    │       ├── generate_volcano_video(提示, 模式, 首帧) → 视频 URL
    │       ├── generate_3d_model(文本/图片)             → OBJ/GLB
    │       ├── qwen_voice_design(音色描述)             → TTS 音频
    │       ├── qwen_omni_understand(媒体文件)          → 理解结果 + 语音
    │       ├── mux_audio_with_bgm(人声, BGM, 参数)    → 混音成品
    │       ├── read_skill_file(SKILL.md)                → Skill 内容
    │       └── write_memory(章节, 内容)                 → 长期记忆
    │
    └── 工具结果: ToolMessage → tool_result 事件
            │
            ├── 图片下载 + sRGB 归一化 + 保存
            ├── 视频流式下载 + 保存
            ├── 3D 模型解压 + 路径修复 + 保存
            └── 音频保存
            │
            ▼
        画布自动插入 / 对话持久化
```

## 训练/执行流程

本项目不涉及模型训练，但 Agent 的推理执行有固定的流程：

1. **环境初始化**：读取 `.env` 配置 → 创建 storage/ 子目录 → 初始化 workspace 默认文件 → 启动 Uvicorn
2. **LLM 工厂创建**：读取 `LLM_PROVIDER` → 实例化对应 Provider (Volcano/SiliconFlow) → 设置 streaming=True, parallel_tool_calls=False, temperature=0.7
3. **Agent 创建** (每次请求)：组装 System Prompt (角色+工具列表+Skills+Workspace) → `create_react_agent(model, tools, prompt)` → 返回编译后的 Agent
4. **消息转换**：前端消息 → LangChain 格式 (HumanMessage / AIMessage)
5. **流式推理**：`agent.astream({"messages": [...]}, stream_mode=["messages"], config={"recursion_limit": 200})`
6. **事件分发**：StreamProcessor 处理每个 chunk → 生成 SSE 事件 → yield 到前端 + WebSocket 广播
7. **工具执行**：Agent 调用 @tool 函数 → 返回 JSON 结果 → StreamProcessor 封装为 tool_result 事件
8. **媒体下载**：工具内部下载生成结果 → 色彩空间转换 → 保存到 storage/ → 返回本地路径
9. **对话持久化**：SSE 流结束后 → HistoryService 保存到 chat_history.json

## FAQ

**Q: Agent 如何决定调用哪个工具？**
A: Agent 基于 ReAct 模式（Reasoning + Acting），每轮推理时 LLM 分析当前上下文决定是否需要工具。System Prompt 中包含完整的工具列表和描述，Agent 根据用户意图自主匹配。例如"生成一张赛博朋克风格的图"会触发 `generate_volcano_image`，"把这个视频和另一个拼接"会触发 `concatenate_videos`。

**Q: Skill 和工具的区别是什么？**
A: 工具是原子操作（生成一张图、合成一段语音），Skill 是领域工作流（论文写作从选题到排版的全流程）。Skill 通过 SKILL.md 定义工作流规范，Agent 加载后按照规范分步执行，每步可能调用多个工具。Skill 采用 Progressive Loading，元数据常驻 System Prompt，完整内容按需加载。

**Q: 如何添加自定义 Skill？**
A: 在 `backend/skills/custom/<skill-id>/` 下创建 SKILL.md（必须包含 YAML frontmatter 中的 name 和 description），然后在设置页启用。Agent 下次对话时自动识别。也可以直接在对话中说"帮我创建一个 XX Skill"，Agent 会通过 init_skill 工具自动初始化目录结构。

**Q: 如何切换 LLM 提供商？**
A: 在 `.env` 中设置 `LLM_PROVIDER=volcano` 或 `LLM_PROVIDER=siliconflow`，重启后端即可。两个提供商通过统一的 `BaseLLMProvider` 接口实现，LLM 工厂根据环境变量自动选择。所有工具与 LLM 解耦——切换 LLM 不影响工具行为。

**Q: 画布数据和聊天记录存储在哪里？**
A: 聊天记录存储于 `backend/storage/chat_history.json`（JSON 数组，每个 canvas 一个条目），生成的媒体文件按类型存储于 `storage/images/`、`storage/videos/`、`storage/audios/`、`storage/models/`。画布 Excalidraw 数据保存于 chat_history.json 的 data 字段。
